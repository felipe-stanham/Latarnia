## What This Skill Does

Pulls the latest version of [ClaudeCodeTemplate](https://github.com/felipe-stanham/ClaudeCodeTemplate) and applies updates to:

- `CLAUDE.md` — full overwrite
- `.claude/skills/` — overwrite per file; add/remove to match template
- `.claude/agents/` — same as skills
- `.claude/hooks/*/template/` — overwrite per file; add/remove to match template
- `.claude/hooks/*/dispatcher.sh` — regenerated every sync

**Never touched:**
- `docs/SYSTEM.md`
- `MEMORY.md`
- `.claude/hooks/*/project/` (your project-specific hook scripts)

---

## Steps

### 1. Fetch the latest template

```bash
TEMPLATE_DIR="$HOME/.claude/ClaudeCodeTemplate"
TEMPLATE_REPO="https://github.com/felipe-stanham/ClaudeCodeTemplate.git"
TEMPLATE_SRC="$TEMPLATE_DIR/Template"

if [ -d "$TEMPLATE_DIR/.git" ]; then
  echo "Updating cached template..."
  git -C "$TEMPLATE_DIR" pull --quiet
else
  echo "Cloning template..."
  git clone --quiet "$TEMPLATE_REPO" "$TEMPLATE_DIR"
fi
```

### 2. Read versions

```bash
CURRENT_VERSION=$(head -5 CLAUDE.md | grep -oE '[0-9]+\.[0-9]+\.[0-9]+' | head -1 || echo "unknown")
INCOMING_VERSION=$(head -5 "$TEMPLATE_SRC/CLAUDE.md" | grep -oE '[0-9]+\.[0-9]+\.[0-9]+' | head -1 || echo "unknown")
echo "Current: v$CURRENT_VERSION → Incoming: v$INCOMING_VERSION"
```

If versions match, report "already up to date" and stop.

### 3. Sync CLAUDE.md

```bash
cp "$TEMPLATE_SRC/CLAUDE.md" CLAUDE.md
echo "✓ CLAUDE.md"
```

### 4. Sync skills

```bash
rsync -a --delete "$TEMPLATE_SRC/.claude/skills/" ".claude/skills/"
echo "✓ .claude/skills/"
```

### 5. Sync agents

```bash
rsync -a --delete "$TEMPLATE_SRC/.claude/agents/" ".claude/agents/"
echo "✓ .claude/agents/"
```

### 6. Sync template hooks and regenerate dispatchers

For each hook type (`pre-tool`, `post-tool`, `stop`):

```bash
DISPATCHER_CONTENT='#!/bin/bash
# AUTO-GENERATED by template-sync — do not edit directly
# To add project-specific hooks, place .sh files in ./project/
set -e
HOOK_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

for f in "$HOOK_DIR/template/"*.sh; do
  [ -f "$f" ] && source "$f"
done

for f in "$HOOK_DIR/project/"*.sh; do
  [ -f "$f" ] && source "$f"
done'

for HOOK_TYPE in pre-tool post-tool stop; do
  DEST=".claude/hooks/$HOOK_TYPE"
  SRC="$TEMPLATE_SRC/.claude/hooks/$HOOK_TYPE/template"

  mkdir -p "$DEST/template" "$DEST/project"

  # Sync template hooks (delete extras)
  if [ -d "$SRC" ]; then
    rsync -a --delete "$SRC/" "$DEST/template/"
  else
    # No template hooks for this type — clear template dir
    find "$DEST/template" -name "*.sh" -delete
  fi

  # Ensure project placeholder exists
  touch "$DEST/project/.gitkeep"

  # Regenerate dispatcher
  echo "$DISPATCHER_CONTENT" > "$DEST/dispatcher.sh"
  chmod +x "$DEST/dispatcher.sh"

  echo "✓ hooks/$HOOK_TYPE/"
done
```

### 7. Report and suggest commit

```
template-sync complete
  v{CURRENT_VERSION} → v{INCOMING_VERSION}
  Updated: CLAUDE.md, .claude/skills/, .claude/agents/, hooks/*/template/, dispatchers
  Skipped (project-owned): docs/SYSTEM.md, MEMORY.md, hooks/*/project/
```

Suggest the user stage and commit:
```
git add -A && git commit -m "chore: sync ClaudeCodeTemplate v{INCOMING_VERSION}"
```

---

## Rules

- Do NOT modify `docs/SYSTEM.md`, `MEMORY.md`, or any file under `hooks/*/project/`.
- If `rsync` is not available, stop and tell the user — do not attempt a manual copy.
- If the clone/pull fails (no network, auth issue), stop and report the error clearly.
- If current version == incoming version, report "already up to date" and make no changes.
