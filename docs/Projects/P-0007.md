- LiteLLM running (`systemctl status latarnia-litellm-tst`)

#### LiteLLMProvisioner module (cap-002)

- [ ] Create `src/latarnia/managers/litellm_provisioner.py` with class `LiteLLMProvisioner(config_manager, secret_manager, env)`.
  - `self.base_url = f"http://{config.host}:{config.port}"`
  - `self.keys_file = Path(config_manager.get_data_dir()).parent.parent / f"{env}" / "litellm_keys.json"` *(verify against `/opt/latarnia/{env}/litellm_keys.json` — the per-env root, not inside `data/`)*
  - `self._key_cache: Dict[str, str]` — loaded from `keys_file` at init.
  - `self._master_key: str` — loaded via `secret_manager.load()` using `config.master_key_secret` as the key name, reading from `/opt/latarnia/{env}/litellm.env` (not `secrets.env`). See note below on `secret_manager` path.
- [ ] Implement `async health_check() -> bool`: `GET /health` with 5-second timeout. Returns `True` if 200, `False` on any error. Log `WARNING` on failure (include URL).
- [ ] Implement `async get_available_models() -> list[str]`: `GET /models`. Parse `response["data"][*]["id"]`. Return `[]` on any error (do not raise). Log `WARNING` on failure.
- [ ] Implement `async provision_key(app_id: str) -> str`:
  - Compute `key = f"sk-latarnia-{self.env}-{app_id}"`.
  - `GET /key/info?key={key}` with `Authorization: Bearer {master_key}`.
  - If 200 → key exists in LiteLLM; skip generate.
  - If 404 (or any error) → `POST /key/generate` with body `{"key": key, "metadata": {"app": app_id, "env": self.env}}`.
  - Persist to `keys_file` (atomic write, mode 600): update `_key_cache` and write JSON.
  - Return `key`.
- [ ] Implement `get_key(app_id: str) -> Optional[str]`: return `_key_cache.get(app_id)`.
- [ ] Implement `async validate_models(app_entry) -> LiteLLMValidationResult`:
  - If `app_entry.manifest.config.requires_models == []` → return `LiteLLMValidationResult(ok=True)`.
  - Call `get_available_models()`.
  - Compute `missing = [m for m in requires_models if m not in available]`.
  - If non-empty → return `LiteLLMValidationResult(ok=False, missing_models=missing, detail=f"model not available in LiteLLM: {missing[0]}")`.
  - Else → `LiteLLMValidationResult(ok=True)`.
- [ ] Implement `async bootstrap_all_keys(registry) -> None`: for each app in `registry.get_all_apps()` where `app.manifest.config.requires_litellm is True`, call `provision_key(app.app_id)`. Log `INFO` with count on completion.
- [ ] Implement `async get_status() -> LiteLLMStatus`: call `health_check()` + `get_available_models()` + read `_key_cache` keys list. Never include key values.
- [ ] **Note on `secret_manager` path for `litellm.env`**: `SecretManager.load()` currently reads from a fixed path (`/opt/latarnia/{env}/secrets.env`). `LiteLLMProvisioner` needs `LITELLM_MASTER_KEY` from `litellm.env`. Either: (a) add a `load_file(path)` method to `SecretManager` for reading arbitrary dotenv files, or (b) inline a minimal dotenv reader in `LiteLLMProvisioner`. Pick whichever is simpler without over-abstracting. Document the choice inline.

### Acceptance Criteria

> Copied verbatim from spec.md. Do not modify.

- **cap-001:** `AppManifest.from_dict({"config": {"requires_litellm": true, "requires_models": ["claude-sonnet"]}})` parses correctly with `config.requires_litellm == True` and `config.requires_models == ["claude-sonnet"]`. `from_dict({"config": {}})` parses with defaults `False` and `[]`. `from_dict({"config": {"requires_models": ["claude-sonnet"]}})` (missing `requires_litellm: true`) raises `ValidationError`.
- **cap-002:** Given a running LiteLLM instance and app `my_app` with `requires_litellm: true`: `LiteLLMProvisioner.provision_key("my_app")` calls `GET /key/info`, gets 404, calls `POST /key/generate` with `key="sk-latarnia-{env}-my_app"`, returns the key string. Second call to `provision_key("my_app")` hits `GET /key/info`, gets 200, skips `POST /key/generate`, returns stored key. `/opt/latarnia/{env}/litellm_keys.json` contains `{"my_app": "sk-latarnia-{env}-my_app"}` after the first call.
- **cap-003:** `ServiceManager.generate_service_template` for an app with `requires_litellm: true` produces a unit containing `Environment=LITELLM_BASE_URL=http://localhost:{port}` and `Environment=LITELLM_API_KEY=sk-latarnia-{env}-{app_name}`. `SubprocessLauncher.start_service` passes the same two env vars in `Popen(env=...)`. Verified via unit tests with mocked LiteLLM client.
- **cap-004:** App with `requires_models: ["claude-opus"]` and LiteLLM returning `models: ["claude-sonnet"]`: `start_service` returns `False`; no unit file written; no port allocated; `app.runtime_info.error_message` contains `"model not available in LiteLLM: claude-opus"`; `GET /api/apps` shows `overall_status: "red"`.
- **cap-005:** Platform startup with LiteLLM unreachable (connection refused): apps with `requires_litellm: true` are not started; `app.runtime_info.error_message` contains `"LiteLLM not reachable"`; other apps (without `requires_litellm: true`) start normally. Platform does not crash.
- **cap-006:** `GET /api/litellm/status` returns 200 with `{"healthy": true|false, "port": <int>, "provisioned_apps": ["app_id_1", ...], "available_models": ["claude-sonnet", ...]}`. `provisioned_apps` lists app IDs with a key in `litellm_keys.json`. `available_models` is empty list (not an error) when LiteLLM is unreachable. No key values appear anywhere in the response.
- **cap-007:** `example_full_app/latarnia.json` has `"requires_litellm": true` and `"requires_models": ["claude-sonnet"]`. On TST with LiteLLM running and `claude-sonnet` configured, `example_full_app` starts successfully and `LITELLM_BASE_URL` + `LITELLM_API_KEY` are present in its environment.

## Migration notes

- **`config.json` update required:** add `"litellm": {"enabled": false}` before deploying. Defaults are safe — existing envs with `enabled: false` are unaffected.
- **Operator bootstrap required before TST verification.** LiteLLM is not managed by the platform; it must be installed and running as a system service before the platform can provision keys. See deployment skill.
- **`example_full_app` manifest change:** adding `requires_litellm: true` means the app will refuse to start in any environment where LiteLLM is not running. On TST, LiteLLM must be bootstrapped before deploying this build.
