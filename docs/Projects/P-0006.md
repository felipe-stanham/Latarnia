## Scope 1: Secret Manager v1

**Branch:** `scope-P-0006-1-secret-manager`
**Status:** `[ ]` → `[IN PROGRESS]` → `[DONE]`
**Depends on:** P-0005 (per-app systemd units, env-scoped paths, `pick_launcher` routing, combined `overall_status` reporting). All landed.

Apps declare `requires_secrets: ["KEY", ...]` in `latarnia.json`. Operator edits `/opt/latarnia/{env}/secrets.env`. The platform validates declared-vs-available, writes a per-app filtered file before launch, references it via `EnvironmentFile=` (Linux) or merges into Popen `env=` (Darwin). Refuses to start when secrets are missing; surfaces in `/api/apps` and on a new `GET /api/secrets` listing endpoint. No values ever appear in logs or REST responses.

#### SecretManager module

- [x] Create `src/latarnia/managers/secret_manager.py` with class `SecretManager(config_manager, app_manager, env)`. Stores `self.master_file_path = Path(config_manager.get_data_dir()).parent / "secrets.env"` *(verify the resolution against `/opt/latarnia/{env}/secrets.env` — the per-env root)*. Stores `self.per_app_dir = self.master_file_path.parent / "secrets"`.
- [x] Implement `_check_mode(path) -> bool` — `os.stat(path).st_mode & 0o077 == 0`. Returns True if mode is sufficiently restrictive (≤600 for files / ≤700 for dirs).
- [x] Implement `_parse_dotenv(text: str) -> Dict[str, str]`:
  - Split by lines.
  - Strip whitespace.
  - Skip blank lines and lines starting with `#`.
  - Parse `KEY=value` (split on first `=`).
  - Support single-quoted values: if `value` starts and ends with `'`, strip them and treat content literally (no escape processing).
  - Reject lines without `=` and lines with empty key (raise `ValueError` so `load()` can log + return `{}`).
- [x] Implement `load() -> Dict[str, str]`:
  - If master file doesn't exist → return `{}` (no warning; valid state for an env with no secrets configured yet).
  - If `_check_mode(master_file_path)` is False → log `WARNING` naming the path + observed octal mode → return `{}`.
  - Otherwise parse the file via `_parse_dotenv` and return the dict.
  - **NEVER log values.** Log line format: `"loaded N secret keys from {path}"` or `"failed to parse secrets file: {error_class}"`.
- [x] Implement `validate(app_entry) -> ValidationResult` (dataclass `ValidationResult(ok: bool, missing: list[str] = [], detail: str = "")`):
  - `declared = app_entry.manifest.config.requires_secrets`.
  - If `declared == []` → return `ValidationResult(ok=True)`.
  - Else: load master, compute `missing = [k for k in declared if k not in master]`.
  - If `missing` non-empty → return `ValidationResult(ok=False, missing=missing, detail=f"missing required secret: {missing[0]}")` (first missing key in detail; full list in `missing` field).
  - Else → return `ValidationResult(ok=True)`.
- [x] Implement `materialize(app_entry) -> Optional[Path]`:
  - Calls `validate(app_entry)`. If `ok=False`, returns `None` (caller already has the result). *(Refactor: have callers receive a single object — propose `validate_and_materialize(app_entry) -> tuple[ValidationResult, Optional[Path]]` or a richer dataclass. Pick what reads cleanest.)*
  - If `declared == []`, returns `None` (no per-app file needed).
  - Else: `mkdir self.per_app_dir mode=0o700` if missing; write `self.per_app_dir / f"{app_id}.env"` with mode 600; content is `f"{k}={v}\n"` per declared key, in declaration order. Use `os.open(...)` with `O_WRONLY | O_CREAT | O_TRUNC` and `mode=0o600` to atomically set perms on creation.
  - Return the per-app file path on success.
- [x] Implement `list_secrets() -> list[SecretMetadata]`:
  - `master = load()`.
  - `set_at = datetime.fromtimestamp(self.master_file_path.stat().st_mtime, tz=timezone.utc)` if file exists else `None`.
  - For each key in `master`: `used_by = [app.app_id for app in app_manager.registry.get_all_apps() if key in (app.manifest.config.requires_secrets or [])]`.
  - Return `list[SecretMetadata(name=k, set_at=set_at, used_by=used_by)]` sorted by name.

### Acceptance Criteria

> Copied verbatim from spec.md. Do not modify. Claude Code's tests must pass against these.

- **cap-001:** `AppManifest.from_dict({"config": {"requires_secrets": ["X", "Y"]}})` parses and `app.manifest.config.requires_secrets == ["X", "Y"]`. `AppManifest.from_dict({"config": {"requires_secrets": "X"}})` (non-list) raises `ValidationError`. `AppManifest.from_dict({"config": {}})` parses with `requires_secrets == []`.
- **cap-002:** Given `/opt/latarnia/dev/secrets.env` with mode 600 containing `KEY1=val1\n# comment\n\nKEY2=val2\n`, `SecretManager(...).load("dev") == {"KEY1": "val1", "KEY2": "val2"}`. Given the same file with mode 644, `load("dev")` returns `{}` and a `WARNING` is logged naming the file path. Given a missing file, `load("dev")` returns `{}` (no warning).
- **cap-003:** Starting `latarnia-tst-myapp.service` where `requires_secrets: ["A", "B"]` and master contains `A=1, B=2, C=3` results in: (a) `/opt/latarnia/tst/secrets/myapp.env` exists with mode 600, content `A=1\nB=2\n` exactly (no `C`); (b) the generated `~felipe/.config/systemd/user/latarnia-tst-myapp.service` contains `EnvironmentFile=-/opt/latarnia/tst/secrets/myapp.env`; (c) `systemctl --user show -p Environment latarnia-tst-myapp.service` shows neither `A` nor `B` (those come from the EnvironmentFile, which `show -p` doesn't expand).
- **cap-004:** On macOS, `SubprocessLauncher.start_service` for the same app passes a `subprocess.Popen` `env=` dict that includes `A=1` and `B=2` but not `C`. Asserted via mocked `Popen` in unit tests.
- **cap-005:** `myapp` with `requires_secrets: ["A", "B"]` and master containing only `A=1`: `start_service("myapp")` returns `False`; no unit file is written; no per-app secrets file is written; `app.runtime_info.error_message` contains `"missing required secret"` and the missing key name; `GET /api/apps` shows `overall_status: "red"` and `overall_status_detail` containing `"B"`. The auto-start loop logs an error but the platform itself continues to boot (other apps unaffected).
- **cap-006:** `GET /api/secrets` returns 200 with body shape `{"secrets": [{"name": "...", "set_at": "<ISO8601>", "used_by": ["app_id_1", ...]}]}`. The list contains exactly the keys present in the current env's master file. `used_by` is the set of registered apps that declare each name in `requires_secrets`. No `value` field appears anywhere. Response is identical regardless of values stored.
- **cap-007:** A unit test sets `secrets.env` with `SENTINEL_KEY=sentinel-value-xyz123`, declares `requires_secrets: ["SENTINEL_KEY"]` for an app, runs through validation + materialize + launch (mocked subprocess), and asserts `caplog.records` contains no record with the substring `xyz123`.
