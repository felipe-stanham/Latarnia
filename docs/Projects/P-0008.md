> the data source: `GET /api/apps` drops apps whose role is `none` for the
> session user (`RoleStore.is_visible`); superusers and unauthenticated dev
> access see all. Role assignment endpoint is superuser-only in full (the

> the role for a machine connection — the webUI header is done here via

## Scope 4: JWT & Machine Tokens

**Branch:** `scope-P-0008-4-jwt-tokens`
**Status:** `[DONE]`
**Depends on:** Scope 2, Scope 3

> **Notes / decisions:** JWT `super` claim mirrors the issuing user's
> `is_superuser` (per data_model) — a superuser token is intentionally
> unscoped (full access); non-superuser tokens are governed by `app_scope`
> (cap-020). Middleware is **pure ASGI** (not BaseHTTPMiddleware) gating
> `/api/*` only, so the mounted `/mcp` SSE app and websockets pass through
> untouched (BaseHTTPMiddleware buffers and would break SSE). MCP auth
> (cap-021) is enforced inside the gateway: Bearer required to open the SSE
> session, tool list scoped via a per-connection ContextVar, and
> `X-Latarnia-App-Role` forwarded to per-app MCP servers (`sse_client(headers=)`).
> cap-019/020 are integration-tested; **cap-021 (live MCP) needs a real MCP
> client and is validated on the manual/tst run** — the JWT-gate, scoping and
> header-injection code paths are in place and unit/logic-tested.

### Tasks
- [x] `src/latarnia/auth/jwt_auth.py` — `JWTAuth.issue`/`validate`/`token_hash` (HS256, `LATARNIA_JWT_SECRET`; claims `{sub, iat, exp, apps, super}`)
- [x] `src/latarnia/auth/tokens.py` — `MachineTokenStore`: create/list/revoke/is_active
- [x] Token endpoints in `auth/routes.py`: `POST` (full gated to superuser; returns raw JWT once), `GET`, `DELETE /{id}` (revoke)
- [x] `src/latarnia/auth/middleware.py` — pure-ASGI `JWTAuthMiddleware` on `/api/*`: Bearer (validate + revocation) or session cookie, else 401; attaches claims to `scope["state"]`
- [x] Applied in `main.py`; `/api/apps` + `/api/apps/{id}` enforce JWT app-scope (cap-020)
- [x] Session + Bearer coexist on `/api/*`
- [x] `mcp_gateway.py`: Bearer-gate the SSE session, scope tool list to claim, inject `X-Latarnia-App-Role` to per-app MCP server (cap-021 — live-validated on tst)
- [x] "API Tokens" dashboard modal: list, new-token form (label, per-app role dropdowns, expiry), revoke, show raw JWT once
- [x] No new DB tables (machine_tokens from Scope 2 migration 005)

### Acceptance Criteria
- **cap-019:** `POST /api/auth/tokens` returns signed JWT; token record in `GET /api/auth/tokens`; `DELETE /api/auth/tokens/{id}` marks revoked; subsequent API call with revoked token → 401
- **cap-020:** `GET /api/apps` with valid JWT → 200; no token → 401; expired JWT → 401; JWT scoped to app A used to access app B's endpoints → 403
- **cap-021:** MCP connection with valid JWT → tool list (scoped to JWT's app_scope); no JWT → 401; per-app MCP server receives `X-Latarnia-App-Role` header matching JWT claim
