# P-0009: App Lifecycle Cleanup

**Created:** 2026-05-18 | **Status:** [ ] Not Started

## Problem

Deleting an app folder leaves orphaned state behind:
- The systemd unit file persists at `~/.config/systemd/user/latarnia-{env}-{app_id}.service`.
- If the unit had `Restart=on-failure/always`, systemd spin-crashes it (working directory gone).
- The port previously held by the app may be re-allocated while the orphan unit still references it.
- There is no dashboard affordance to stop + clean up + delete an app in one action.

## Scope 1: Backend Cleanup

**Branch:** `scope-P-0009-1-backend-cleanup`
**Status:** `[ ]` → `[IN PROGRESS]` → `[DONE]`
**Depends on:** none

### Tasks
- [ ] Add `AppManager.get_orphaned_apps()` — returns registry entries whose `path` no longer exists on disk
- [ ] Add `_cleanup_orphaned_apps()` async helper in `main.py` — stops service via `pick_launcher`, calls `service_manager.remove_service`, releases ports, calls `app_manager.unregister_app`
- [ ] Call `_cleanup_orphaned_apps()` in lifespan startup (after `discover_apps`)
- [ ] Call `_cleanup_orphaned_apps()` in `POST /api/apps/discover` endpoint (add `orphans_cleaned` to response)
- [ ] Extend existing `DELETE /api/apps/{app_id}` endpoint: stop service, remove systemd unit, release ports, unregister, and (if `delete_folder=true` query param) `shutil.rmtree` the app path

### Acceptance Criteria
- **orphan-detect:** When `POST /api/apps/discover` is called and a registered app's folder is absent, the app is unregistered and its systemd unit is stopped + removed. The response includes `orphans_cleaned`.
- **delete-endpoint:** `DELETE /api/apps/{app_id}?delete_folder=false` stops the service and removes the unit but leaves the folder. `delete_folder=true` also removes the folder.
- **port-release:** After deletion, the app's REST and MCP ports are available for allocation to other apps.
- **no-crash:** Deleting an app that is already stopped (or whose unit doesn't exist) completes without error.

### Acceptance Criteria
- **button-visible:** "Delete App" button appears in the app details modal for all app types.
- **confirmation:** Clicking Delete shows a confirmation dialog listing what will happen; cancelling aborts.
- **success-flow:** On successful delete, the modal closes and the app card disappears from the dashboard without a full page reload.
- **error-display:** If the API returns an error, an alert is shown and the modal stays open.
