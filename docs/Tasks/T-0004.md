# T-0004: Add `public_routes` manifest field + Caddyfile generator support

**Status:** [ ] Open  
**Branch:** `task-T-0004-public-routes`  
**Created:** 2026-06-12  

## Context

Okno (and any future app that serves unauthenticated public content) needs certain path prefixes to bypass Caddy's `forward_auth`. Today, the only auth-exempt per-app paths are `/apps/{name}/docs*` and `/apps/{name}/openapi.json` (P-0008 cap-005), hardcoded in the Caddyfile generator (`caddy/manager.py generate_config()`, ~lines 186‚Äì200).

`docs/System/app-specification.md` now documents `/b/` as the reserved public-bundle prefix (convention), but there is no mechanism for an app to declare that its routes under `/b/` (or any other prefix) should be served without auth.

**Path semantics (critical):** the protected app block is `handle_path /apps/{name}/*`, which strips `/apps/{name}` so the app receives `/b/test`. A public block must reproduce exactly that stripping. A plain `handle` (no strip) would deliver `/apps/{name}/b/test` ‚Üí app 404s; `handle_path /apps/{name}/b/*` would over-strip to `/test`. The correct shape is `handle` + explicit `uri strip_prefix`.

**Header spoofing (critical):** blocks without `forward_auth` pass client-supplied headers through, and apps trust `X-Latarnia-User` / `X-Latarnia-App-Role` / `X-Latarnia-Is-Super`. Every no-auth app block must delete incoming `X-Latarnia-*` request headers. (The existing per-app `/docs*` / `openapi.json` public blocks have this hole today ‚Äî fix them in the same change.)

## Scope

Add `public_routes` as an optional manifest field and wire it through the Caddyfile generator so declared prefixes are served without `forward_auth`, with identity headers stripped.

Promotion triggers re-checked at spec time: touches the App Manager (manifest parsing) and the Caddy generator, but it is one bounded scope, no `dataModel.md` change, no new external service ‚Äî staying a Task.

## Implementation Plan

- [ ] **1. Manifest field** ‚Äî add `public_routes` (array of path-prefix strings, default `[]`) to the `AppConfig` Pydantic model in `app_manager.py` (where `has_web_ui` lives). Pydantic ignores unknown fields here, so the addition is non-breaking; log the parsed value at discovery (DEBUG) so a typo'd field name is noticeable.

- [ ] **2. Validation** ‚Äî at discovery, drop (log WARNING, do not fail the app) any entry that: does not start with `/`; is exactly `/` (would make the whole app public); is empty; or overlaps a reserved path (`/health`, `/docs`, `/openapi.json`).

- [ ] **3. App Registry Entry** ‚Äî `public_routes` rides along on the registry entry's `manifest.config`, same as `has_web_ui` ‚Äî confirm the Caddy generator can read it from `_web_ui_apps()` / the registry without a new field on `AppRegistryEntry`.

- [ ] **4. Caddyfile generator** ‚Äî in `generate_config()`, for each app with `has_web_ui: true` and non-empty `public_routes`, emit per declared prefix, **before** the app's protected `handle_path /apps/{name}/*` block:

  ```
  handle /apps/{name}{prefix}* {
      uri strip_prefix /apps/{name}
      reverse_proxy localhost:{port} {
          header_up -X-Latarnia-User
          header_up -X-Latarnia-App-Role
          header_up -X-Latarnia-Is-Super
      }
  }
  ```

  No `forward_auth`. Note Caddy also sorts same-name directives by path-matcher length (longest first), so the longer public prefix wins over `/apps/{name}/*` regardless ‚Äî but keep emission order specific-first anyway, matching the existing docs-block pattern. Verify behavior with curl, not just by reading the Caddyfile.

- [ ] **5. Strip identity headers on the existing public blocks** ‚Äî add the same `header_up -X-Latarnia-*` deletions to the `/apps/{name}/docs*` and `/apps/{name}/openapi.json` blocks (pre-existing spoofing hole).

- [ ] **6. Reload on change** ‚Äî `public_routes` changes take effect the same way any other manifest change does: rediscover the app (manual refresh or restart). No special handling needed.

- [ ] **7. Unit tests** ‚Äî extend `tests/unit/test_caddy_manager.py`: public block emitted with `uri strip_prefix` and header deletions and without `forward_auth`; no block for apps with absent/empty `public_routes`; invalid entries (`/`, no leading `/`, reserved overlap) skipped with a warning.

- [ ] **8. Update `example_full_app`** ‚Äî add `"public_routes": ["/b/"]` to its manifest and a stub `GET /b/test` route that returns 200 **and echoes any received `X-Latarnia-*` headers in its JSON body** (lets the acceptance test prove header stripping). Update `examples/`, never `apps/`.

- [ ] **9. Update `app-specification.md`** ‚Äî mark `public_routes` as shipped (remove the "not yet shipped" note) and document the manifest field under Optional Fields, including the validation rules from step 2.

## Acceptance Criteria

- `example_full_app` manifest declares `"public_routes": ["/b/"]`.
- `GET /apps/example_full_app/b/test` returns 200 **without** a session cookie on TST (proves prefix stripping delivered `/b/test` to the app).
- `GET /apps/example_full_app/b/test` with header `X-Latarnia-Is-Super: true` ‚Üí the app's echo shows **no** `X-Latarnia-*` headers received (proves stripping).
- `GET /apps/example_full_app/` still redirects to `/auth/login` without a session cookie.
- Generated Caddyfile contains a `handle /apps/example_full_app/b/*` block with `uri strip_prefix /apps/example_full_app`, the three `header_up -X-Latarnia-*` deletions, and no `forward_auth`; `caddy validate` accepts it.
- A manifest with `"public_routes": ["/"]` (or a non-`/`-prefixed entry, or `/docs`) ‚Üí entry skipped with a WARNING log; app still starts; Caddyfile unchanged for that entry.
- Apps without `public_routes` (or with `[]`) generate a Caddyfile identical to today **except** the header deletions added to the existing docs/openapi public blocks ‚Äî no other regression.

## Notes

- `/b/` is the canonical convention but the field accepts any prefix ‚Äî keep it general.
- **Trust model:** a public route is fully anonymous ‚Äî the app receives no identity headers there. Apps must not gate logic on `X-Latarnia-*` under their `public_routes` prefixes.
- **Merge coordination:** P-0010 Scope 1 also edits `caddy/manager.py` (`_forward_auth_lines`) and `test_caddy_manager.py`. Land this task and that scope sequentially off `dev`, not in parallel.
